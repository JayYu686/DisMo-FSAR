Generate one JSON object for the action class "{class_name}".

Return JSON only. No markdown fences. No extra prose.

Schema:
{
  "label_text": "canonical short class phrase",
  "action_anchor": "one short visual sentence",
  "entity_priors": ["entity 1", "entity 2"],
  "scene_priors": ["scene cue 1"],
  "attribute_cues": ["attribute cue 1", "attribute cue 2"],
  "phase_cues": [
    "phase 1 short visual cue",
    "phase 2 short visual cue",
    "phase 3 short visual cue"
  ],
  "confusion_cues": ["easy confusion 1", "easy confusion 2"]
}

Hard rules:
1) English only.
2) Preserve placeholders like [something], [part], [number of], [somewhere] exactly.
3) SSV2 is not about rich background descriptions. Do not mention kitchen, room, indoor, dining area, counter, tabletop, home, or office.
4) Make attribute_cues and phase_cues the priority. They matter more than scene_priors.
5) scene_priors should usually be [].
6) entity_priors should contain at most 2 essential entities, usually hand, object, container, surface, camera, or stack.
7) attribute_cues must be 2-4 short canonical cues from these types:
   direction, spatial relation, contact, support, containment, state change, motion property, pretend/fail.
8) phase_cues must be exactly {num_phases} short cues:
   initial state -> main change -> final state.
9) phase_cues must be short phrases, not long captions.
10) For pretend classes, the last phase must show that no real result happens.
11) For fail classes, the last phase must show the target state is not reached.
